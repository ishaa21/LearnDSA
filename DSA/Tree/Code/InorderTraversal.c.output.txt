=== Inorder Traversal (C) ===

Tree structure:
         1
        / \
       2   3
      / \ / \
     4  5 6  7

Recursive Inorder (Left-Root-Right): 4 2 5 1 6 3 7
Iterative Inorder (using stack):     4 2 5 1 6 3 7

=== Inorder Characteristics ===
1. Left subtree → Root → Right subtree
2. For BST: Produces sorted sequence
3. Recursive approach: Clean and simple
4. Iterative approach: Explicit stack management

=== Complexity Analysis ===
Time Complexity:  O(n) - visit each node once
Space Complexity: O(h) - h = height (recursion/stack)
                   O(n) worst case (skewed tree)
