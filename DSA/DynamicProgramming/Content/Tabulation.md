# Tabulation

## What is this?
Bottom-up DP approach.

## Why do we need it?
Avoid recursion stack.
Faster execution.

## Algorithm Logic
Fill DP table iteratively.

## Common Mistakes
Wrong table order.

## Time Complexity
O(n)

## Space Complexity
O(n) - can sometimes be O(1) if optimized.

## Real Life Use
Optimizing iterative calculations.

## Key Takeaways
- Iterative DP
- Can save memory
